1. Overview

Content Hub is a full-stack blogs and articles platform. The React frontend is hosted on Vercel; the Spring Boot API runs on Render (Docker); data lives in Supabase PostgreSQL.

flowchart LR
  Dev[Developer] --> FE[Vercel Frontend]
  Dev --> API[Render API]
  FE -->|HTTPS REST| API
  API --> DB[(Supabase PostgreSQL)]
  Dev -->|OAuth| Google[Google Identity]
  Google --> API

3. Local Development Setup

Prerequisites

  • Node.js 20+, Java 17, Docker (optional local Postgres)
  • Supabase project or local docker compose database

Quick start

git clone https://github.com/AbhishekMD1998/content_hub.git
cd content-hub
bash scripts/setup.sh          # installs deps, Maven wrapper
docker compose up -d           # local Postgres on :5433 (optional)
bash scripts/configure-supabase.sh   # OR copy backend/.env.example
bash scripts/start-backend.sh  # API on http://localhost:8080
npm run dev                    # Frontend on http://localhost:5173
flowchart TD
  A[Clone repo] --> B[setup.sh]
  B --> C{Database?}
  C -->|Local| D[docker compose up]
  C -->|Cloud| E[configure-supabase.sh]
  D --> F[start-backend.sh]
  E --> F
  F --> G[npm run dev]
  G --> H[Open localhost:5173]
In dev, VITE_API_BASE_URL is unset — Vite proxies /api, /oauth2, and /login to port 8080.

6. Authentication Flows

6.1 Email / Password Login

sequenceDiagram
  participant U as User Browser
  participant FE as React App
  participant API as Spring Boot API
  participant DB as PostgreSQL

  U->>FE: Submit login form
  FE->>API: POST /api/auth/login
  API->>DB: Find user by email
  DB-->>API: User + password_hash
  API->>API: BCrypt verify + JWT sign
  API-->>FE: { token, user }
  FE->>FE: localStorage content-hub-token
  FE-->>U: Redirect to /admin

6.2 Google OAuth

sequenceDiagram
  participant U as User
  participant FE as Vercel Frontend
  participant API as Render API
  participant G as Google
  participant DB as Supabase

  U->>FE: Click Continue with Google
  FE->>API: GET /oauth2/authorization/google
  API->>G: OAuth redirect
  G-->>U: Consent screen
  U->>G: Approve
  G->>API: GET /login/oauth2/code/google
  API->>DB: findOrCreateGoogleUser
  API-->>FE: Redirect /auth/callback?token=JWT
  FE->>API: GET /api/auth/me
  FE-->>U: Redirect /admin

6.3 Protected API requests

flowchart LR
  REQ[HTTP Request] --> FILTER[JwtAuthenticationFilter]
  FILTER -->|Bearer valid| CTX[SecurityContext]
  FILTER -->|No/invalid token| ANON[Anonymous]
  CTX --> CTRL[Controller]
  ANON --> PUB{Public endpoint?}
  PUB -->|Yes| CTRL
  PUB -->|No| DENY[401 Unauthorized]

11. Troubleshooting

IssueFix
Cannot reach API from VercelSet VITE_API_BASE_URL on Vercel
Google redirect_uri_mismatchAdd exact Render callback URL in Google Console
Render Network unreachableUse Supabase session pooler, not direct db.* host
API slow first requestRender free tier cold start (~60–120s)
CORS errorsMatch FRONTEND_URL in Render CORS_ALLOWED_ORIGINS
