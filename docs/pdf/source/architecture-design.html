1. Executive Summary

Content Hub is a three-tier web application delivering a public content portal (blogs and articles) with an authenticated admin publishing workflow. The architecture separates static UI delivery (Vercel CDN), stateful API logic (Render containers), and managed persistence (Supabase PostgreSQL).

GoalApproach
Fast global UIStatic React SPA on Vercel edge CDN
JVM backend without PaaS Java supportDocker on Render
Managed PostgreSQLSupabase with Flyway migrations
Flexible authJWT + optional Google OAuth2
Cost-effective MVPFree tiers: Vercel, Render, Supabase

Schema migration strategy

Flyway manages schema via V1__init_schema.sql. baseline-on-migrate: true allows coexistence with manually applied Supabase SQL. Hibernate ddl-auto: validate ensures entity/schema alignment at runtime.

JWT token structure

stateDiagram-v2
  [*] --> Anonymous
  Anonymous --> Authenticated: login / signup / Google OAuth
  Authenticated --> Anonymous: logout (clear token)
  Authenticated --> Authenticated: API calls with Bearer token
  Anonymous --> Anonymous: browse public content

Blog rendering

  • JSON blogs (seeded): BlogRenderer.jsx renders structured blocks (headings, paragraphs, stickers, TOC)
  • Admin blogs: plain text content with paragraph breaks
  • Articles: read-only curated content, seeded on first boot

9. API Design Principles

  • RESTful resources — nouns (/api/blogs), HTTP verbs for actions
  • Slug-based identity — human-readable URLs for blogs and articles
  • Stateless sessions — JWT in Authorization header, no server-side session store
  • Consistent errors — { "message": "..." } via ApiExceptionHandler
  • Public read, authenticated write — content discovery without login
